#### 1.5 环境配置

说明：这里的环境配置主要配置的是npm安装的全局模块所在的路径，以及缓存cache的路径，之所以要配置，是因为以后在执行类似：npm install express [-g] （后面的可选参数-g，g代表global全局安装的意思）的安装语句时，会将安装的模块安装到【C:\Users\用户名\AppData\Roaming\npm】路径中，占C盘空间。

例如：我希望将全模块所在路径和缓存路径放在我node.js安装的文件夹中，则在我安装的文件夹【D:\DevSoft\nodejs】下创建两个文件夹【node_global】及【node_cache】如下图：

![011](/documents/img/guide/notes/011.png)

创建完两个空文件夹之后，打开 cmd 命令窗口，输入以下命令（注意：修改成自己的目录）
```
npm config set prefix "D:\DevSoft\nodejs\node_global"

npm config set cache "D:\DevSoft\nodejs\node_cache"
```

接下来设置环境变量，关闭 cmd 窗口，“我的电脑”-右键-“属性”-“高级系统设置”-“高级”-“环境变量”

- Windows10

![012](/documents/img/guide/notes/012.png)

- Windows11

![013](/documents/img/guide/notes/013.png)

进入环境变量对话框，在【系统变量】下新建【NODE_PATH】，输入
【D:\DevSoft\nodejs\node_global\node_modules】`注意：按照自己的路径写`，将【用户变量】下的【Path】修改为【D:\DevSoft\nodejs\node_global】

### 二、 安装git

#### 2.1 安装

本机安装好git（详情参考文档） [Git下载安装教程：git安装步骤手把手图文【超详细】](https://zhuanlan.zhihu.com/p/443527549)

安装完成后，还需要最后一步设置（Git 全局设置--提交时会引用），在命令行输入：
```
$ git config --global user.name "Your Name"
$ git config --global user.email "email@example.com"
```
> 说明：
因为 Git 是分布式版本控制系统，所以，每个机器都必须自报家门：你的名字和 Email 地址。
`git config` 命令的 `--global` 参数，用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置，当然也可以对某个仓库指定不同的用户名和 Email 地址

#### 2.2 命令行常用指令

```
git clone [remote_url]      //克隆远程仓库项目到本地当前目录
git pull                    //拉取服务器代码
git init                    //初始化
git add .                   //将状态改变的代码提交至缓存
git commit -m “注释部分”     //将代码提交到本地仓库中
git remote add origin [url] //添加远程仓库
git push -u origin master   //将代码推送到服务器master分支，-u参数指定一个默认主机
```

#### 2.3 git提交代码步骤

`git clone` -> 工作区 

`git add` -> 暂存区 

`git commit` -> 版本库 

`git push` -> 远程自己的仓库

## 开始工作

### 下载源码

在自己的指定目录下鼠标右键，选择`Git Bash Here`，执行命令

```
git clone https://github.com/chinaguocg/documents.git

# OR git clone git@github.com:chinaguocg/documents.git
```

### 安装依赖与启动

在工程目录`documents`下，执行如下命令

- 安装依赖

```
npm install
```

- 启动

```
npm run docs:dev
```
