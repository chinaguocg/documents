# 快速开始

# 准备工作

注意

如已安装 Node.js 和 Git ，请忽略此项，直接进行下一步 开始工作

# 一、 安装node.js

# 1.1 安装环境

  • 操作系统：Windows 10 Enterprise、Windows 11 Enterprise

  • Node.js：v16.13.1（64位）

# 1.2 安装Node.js步骤

  • 下载对应自己系统的 Node.js 版本:https://nodejs.org/en/download/ (opens new window)

  • 选安装目录进行安装

  • 环境配置

  • 测试

# 1.3 前期准备

  • 1.3.1 Node.js简介

简单的说 Node.js 就是运行在服务端的 JavaScript。Node.js 是一个基于 Chrome V8 引擎的 JavaScript 运行环境。Node.js 使用了一个事件驱动、非阻塞式 I/O 的模型，使其轻量又高效。Node.js 的包管理器 npm，是全球最大的开源库生态系统。

  • 1.3.2 下载Node.js

打开官网下载链接:https://nodejs.org/en/download/ (opens new window)，可以直接下载最新版本

也可以选择历史版本进行下载，点击 Previous Releases (opens new window) 选择历史版本，如下图

此处以 v16.13.1（64位 Windows操作系统） 为例，如下图

# 1.4 开始安装

下载完成后，双击node-v16.13.1-x64.msi，开始安装 Node.js ，点击【Next】

直到进入安装目录的设置界面，可以自定义安装目录后，再点击【Next】按钮

点击【Next】，待设置完成后，点击【Install】按钮

安装完后点击【Finish】按钮完成安装

至此 Node.js 已经安装完成，可以先进行下简单的测试安装是否成功了，后面还要进行环境配置 在键盘按下【win+R】键，输入cmd，然后回车，打开cmd窗口

# 1.5 环境配置

说明：这里的环境配置主要配置的是npm安装的全局模块所在的路径，以及缓存cache的路径，之所以要配置，是因为以后在执行类似：npm install express [-g] （后面的可选参数-g，g代表global全局安装的意思）的安装语句时，会将安装的模块安装到【C:\Users\用户名\AppData\Roaming\npm】路径中，占C盘空间。

例如：我希望将全模块所在路径和缓存路径放在我node.js安装的文件夹中，则在我安装的文件夹【D:\DevSoft\nodejs】下创建两个文件夹【node_global】及【node_cache】如下图：

创建完两个空文件夹之后，打开 cmd 命令窗口，输入以下命令（注意：修改成自己的目录）

npm config set prefix "D:\DevSoft\nodejs\node_global"

npm config set cache "D:\DevSoft\nodejs\node_cache"

接下来设置环境变量，关闭 cmd 窗口，“我的电脑”-右键-“属性”-“高级系统设置”-“高级”-“环境变量”

  • Windows10

  • Windows11

进入环境变量对话框，在【系统变量】下新建【NODE_PATH】，输入 【D:\DevSoft\nodejs\node_global\node_modules】注意：按照自己的路径写，将【用户变量】下的【Path】修改为【D:\DevSoft\nodejs\node_global】

# 1.6 测试

配置完后，安装个 module 测试下，我们就安装搭建该文档所依赖的 vuepress 模块，打开 cmd 窗口， 输入如下命令进行模块的全局安装：

npm install -g vuepress
1

说明： -g 是全局安装的意思

安装完成后，可以看到 vuepress 模块安装在刚才配置的全局模块路径的文件夹中

# 二、 安装git

# 2.1 安装

本机安装好git（详情参考文档） Git下载安装教程：git安装步骤手把手图文【超详细】 (opens new window)

安装完成后，还需要最后一步设置（Git 全局设置--提交时会引用），在命令行输入：

$ git config --global user.name "Your Name"
$ git config --global user.email "email@example.com"
1
2

说明： 因为 Git 是分布式版本控制系统，所以，每个机器都必须自报家门：你的名字和 Email 地址。 git config 命令的 --global 参数，用了这个参数，表示你这台机器上所有的Git仓库都会使用这个配置，当然也可以对某个仓库指定不同的用户名和 Email 地址

# 2.2 命令行常用指令

git clone [remote_url]      //克隆远程仓库项目到本地当前目录
git pull                    //拉取服务器代码
git init                    //初始化
git add .                   //将状态改变的代码提交至缓存
git commit -m “注释部分”     //将代码提交到本地仓库中
git remote add origin [url] //添加远程仓库
git push -u origin master   //将代码推送到服务器master分支，-u参数指定一个默认主机
